# Project Bedrock — InnovateMart EKS Deployment

**Karatu 2025 Capstone | Cloud DevOps Engineering**

A production-grade microservices deployment of the **AWS Retail Store Sample App** on Amazon EKS, provisioned entirely with Terraform, with managed AWS data services, observability, secure developer access, an event-driven serverless pipeline, and a CI/CD workflow.

---

## Application

This project deploys the official **[`aws-containers/retail-store-sample-app`](https://github.com/aws-containers/retail-store-sample-app)** — AWS's reference retail store microservices application (product catalog, shopping cart, checkout, orders, and UI).

- **Deployment method:** Helm chart `retail-store-sample-chart` **v0.8.5** (vendored in `kubernetes/retail-store-sample-chart-0.8.5.tgz`).
- **Data layer override:** the chart's default in-cluster databases are **disabled** and replaced with managed AWS services:
  - `catalog` → **Amazon RDS (MySQL)**
  - `orders` → **Amazon RDS (PostgreSQL)**
  - `carts` → **Amazon DynamoDB**
  - `checkout` → Redis (in-cluster, per chart default — allowed)
- Database credentials are generated by Terraform and stored in **AWS Secrets Manager**, injected at deploy time — never hardcoded.

**Live Store URL:** `http://k8s-retailap-retailst-17d19cf248-1977344716.us-east-1.elb.amazonaws.com`

## Architecture

```
Internet
   │
   ▼
Application Load Balancer (internet-facing)  ──  AWS Load Balancer Controller (IRSA)
   │
   ▼
EKS Cluster: project-bedrock-cluster (v1.34, 3× t3.small, private subnets)
   ├── ui ─────▶ catalog ─▶ RDS MySQL
   │            ├ carts  ─▶ DynamoDB (IRSA-scoped access)
   │            ├ orders ─▶ RDS PostgreSQL
   │            └ checkout ─▶ Redis (in-cluster)
   │
   └── Observability: CloudWatch Observability add-on (FluentBit) ─▶ CloudWatch Logs

Event-driven: S3 (assets bucket) ──ObjectCreated──▶ Lambda (bedrock-asset-processor) ──▶ CloudWatch Logs
State & CI/CD: Terraform remote state in S3 (native locking) | GitHub Actions: plan on PR, apply on merge
```

All resources are in **us-east-1** and tagged `Project: karatu-2025-capstone`.

## Deployment Guide

### Prerequisites
AWS CLI (configured), Terraform ≥ 1.10, kubectl, helm, eksctl.

### 1. Provision infrastructure
```bash
cd terraform
terraform init
terraform apply
terraform output -json > ../grading.json
```
Remote state is stored in S3 bucket `bedrock-tfstate-dibia-2026` with native S3 state locking.

### 2. Connect to the cluster
```bash
aws eks update-kubeconfig --name project-bedrock-cluster --region us-east-1
kubectl get nodes
```

### 3. Install the AWS Load Balancer Controller (IRSA)
```bash
eksctl utils associate-iam-oidc-provider --cluster project-bedrock-cluster --region us-east-1 --approve
helm repo add eks https://aws.github.io/eks-charts && helm repo update
helm install aws-load-balancer-controller eks/aws-load-balancer-controller \
  -n kube-system --set clusterName=project-bedrock-cluster \
  --set serviceAccount.create=false --set serviceAccount.name=aws-load-balancer-controller
```

### 4. Deploy the application (single command)
```bash
kubectl create namespace retail-app
# DB passwords pulled from Secrets Manager and injected at install (never committed):
helm upgrade --install retail-store kubernetes/retail-store-sample-chart-0.8.5.tgz \
  -n retail-app -f kubernetes/values.yaml \
  --set "catalog.mysql.secret.password=$MYSQL_PW" \
  --set "orders.postgresql.secret.password=$PG_PW" \
  --wait --timeout 10m
kubectl apply -f kubernetes/ingress.yaml
```

### 5. Get the Store URL
```bash
kubectl get ingress -n retail-app   # ADDRESS column = live ALB URL
```

## Secure Developer Access

`bedrock-dev-view` IAM user (defined in `terraform/iam.tf`):
- **Console:** `ReadOnlyAccess` managed policy.
- **Kubernetes:** mapped to `AmazonEKSViewPolicy` via EKS access entry (read-only).
- **S3:** scoped `s3:PutObject` on the assets bucket only.

Verified: `kubectl get pods -n retail-app` succeeds; `kubectl delete pod` returns `Forbidden`.

---

## Observability

- **Control plane logs:** all five log types → CloudWatch `/aws/eks/project-bedrock-cluster/cluster`.
- **Container logs:** `amazon-cloudwatch-observability` add-on (FluentBit) → `/aws/containerinsights/project-bedrock-cluster/application`.

---

## Event-Driven Serverless

Uploading a file to the S3 assets bucket triggers the `bedrock-asset-processor` Lambda (via S3 Event Notification), which logs `Image received: [filename]` to CloudWatch.

---

## CI/CD (GitHub Actions)

- **Pull Request** → `terraform plan` (posted as PR comment).
- **Merge to `main`** → `terraform apply`.
- AWS credentials stored as repository secrets (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`) for a dedicated `bedrock-cicd` IAM user — never hardcoded.
